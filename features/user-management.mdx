## Filtering users

A search bar and two filter controls sit above the user list. All three operate simultaneously â€” only users that match every active filter are shown.

### Search bar

The text input matches against three fields in real time:

- Full name (`nombreCompleto`)
- Email address (`email`)
- Role (`rol`)

### Status filter

Three toggle buttons let you narrow the list by account status:

| Button | Behaviour |
|---|---|
| **Todos** | Shows all users (default) |
| **Activos** | Shows only users where `isActive` is `true` |
| **Inactivos** | Shows only users where `isActive` is `false` |

The active button is highlighted in navy (`#0F2C59`).

### Department dropdown

A `<select>` element is populated dynamically from the unique department values present in the current user list. Selecting a department limits the list to users assigned to that department. The default option is **Todos los departamentos**.

### API payload

The `useUsuarios` hook sends the following JSON body to `POST /api/v1/usuarios`:

```javascript
{
  nombre: formData.nombre,
  apellidoPaterno: formData.apellidoPaterno,
  apellidoMaterno: formData.apellidoMaterno,
  correo: formData.correo,
  telefono: formData.telefono,
  horaEntrada: `${formData.horaEntrada}:00`,   // e.g. "08:00:00"
  horaSalida: `${formData.horaSalida}:00`,      // e.g. "16:00:00"
  roles: formData.roles,                         // array of role strings
  departamentoId: Number(formData.departamentoId)
}
```

Note that seconds (`:00`) are appended to both time values before the request is sent.

## Editing a user

Click the **pencil icon** button on any user card to open the same modal in edit mode. The modal title changes to **Editar Usuario** and all fields are pre-filled with the existing account data:

- The full name is split on spaces to populate `nombre`, `apellidoPaterno`, and `apellidoMaterno`
- `horaEntrada` and `horaSalida` are trimmed to `HH:MM` (the trailing `:00` is stripped)
- `roles` is populated from `rolesOriginales` if available, otherwise from the normalised `rol` field

<Warning>
The PUT endpoint is not yet connected to the backend. Submitting the edit form currently logs the payload to the console and shows a confirmation alert. The user list is **not** refreshed after an edit until the PUT integration is complete.
</Warning>

## Activating and deactivating accounts

Each user card displays either a **Desactivar** button (for active users, red border) or an **Activar** button (for inactive users, green border).

Clicking either button opens a confirmation modal before any change is made.

### Confirmation modal behaviour

| Action | Modal type | Icon | Confirm button |
|---|---|---|---|
| Deactivate | `danger` | Red alert circle | Red "Desactivar" button |
| Activate | `success` | Green check circle | Green "Activar" button |
