JyPS is a single-page application that digitizes the workflow for requesting, approving, and validating employee exit passes and absence justifications. It supports five distinct user roles, real-time filtering, and QR-based access control for security checkpoints.

## What JyPS does

JyPS replaces paper-based processes with a structured digital workflow:

<Steps>
  <Step title="Employee submits a request">
    An employee logs in and requests an exit pass (pase de salida) or submits an absence justification (justificante). Each request includes the date, time, and reason.
  </Step>
  <Step title="Manager or HR reviews the request">
    The area manager (jefe de área) sees pending requests on their dashboard and approves or rejects each one. HR reviews justifications for payroll purposes.
  </Step>
  <Step title="Approved request generates a QR code">
    Once approved, the system generates a unique QR code that the employee presents at the security checkpoint.
  </Step>
  <Step title="Security guard validates the code">
    The security guard scans or manually enters the code to register the exit and, later, the return of the employee.
  </Step>
</Steps>

## Key features

<CardGroup cols={2}>
  <Card title="Role-based panels" icon="layout-dashboard" href="/roles/overview">
    Five dedicated dashboards — Administrator, Area Manager, HR, Employee, and Security Guard — each showing only what that role needs.
  </Card>
  <Card title="QR code validation" icon="qrcode" href="/features/qr-validation">
    Security guards validate exit passes by scanning or typing a unique QR code, with real-time exit and return logging.
  </Card>
  <Card title="Approval workflows" icon="check-circle" href="/features/approvals">
    Pending requests are queued by submission time so managers can act on the oldest requests first.
  </Card>
  <Card title="Request history" icon="clock-rotate-left" href="/features/request-history">
    Every user can view the full history of their own requests and track approval status.
  </Card>
</CardGroup>
