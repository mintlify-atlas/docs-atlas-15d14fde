## Three-layer architecture

JyPS separates responsibilities into three layers that map directly onto the directory structure:

<CardGroup cols={3}>
  <Card title="View layer" icon="layout">
    **`pages/` and `components/`**

    Pure presentation. No direct data fetching, no `fetch()` calls. Pages receive data via props or by calling domain hooks. Components are stateless UI primitives.
  </Card>

  <Card title="Domain layer" icon="code">
    **`hooks/useUsuarios.js`**

    Business logic and data mapping. Translates raw backend responses into clean shapes the view layer expects. Wraps infrastructure calls and returns `{ exito, data, mensaje }`.
  </Card>

  <Card title="Infrastructure layer" icon="server">
    **`hooks/useApi.js`**

    HTTP transport. Manages the `fetch` lifecycle, sets headers, handles `isLoading` and `error` state, normalizes HTTP error responses, and exposes a single `request()` function.
  </Card>
</CardGroup>

## Routing

Routes are defined in a single file (`src/routes.jsx`) using React Router 7's `createBrowserRouter`. Each role gets its own path prefix with a layout component that renders shared chrome (nav, sidebar) and an `<Outlet />` for nested pages.

| Path prefix | Role | Layout component |
|-------------|------|-----------------|
| `/` or `/login` | Unauthenticated | — |
| `/administrador` | Administrator | `AdministradorLayout` |
| `/trabajador` | Employee | `TrabajadorLayout` |
| `/recursos-humanos` | HR | `RRHHLayout` |
| `/jefe-area` | Area manager | `JefeLayout` |
| `/guardia` | Security guard | `GuardiaLayout` |

Pages under `globales/` (`Perfil`, `Historial`) are mounted as children of **every** role layout, so they are accessible from any panel without duplicating page code.

## Entry point

`src/main.jsx` bootstraps the React tree using the React 19 `createRoot` API:

```javascript src/main.jsx
import { StrictMode } from 'react'
import { createRoot } from 'react-dom/client'
import './index.css'
import App from './App.jsx'

createRoot(document.getElementById('root')).render(
  <StrictMode>
    <App />
  </StrictMode>,
)
```

`App.jsx` renders the `RouterProvider` backed by the router exported from `routes.jsx`.
